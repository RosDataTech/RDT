# Документация по работе с `main.py`

В этом разделе описаны основные функции и структуры, используемые в `main.py`. Неиспользуемые напрямую вспомогательные функции опущены для краткости.

---

## Модуль: `rdt_pdf` as `pdf`

### `pdf.uni_index`
Cловарь индексации университетов.

### `pdf.extract_text(pdf_url)`
Парсит весь текст из PDF по указанному URL и возвращает его в виде строки.

**Параметры:**
- `pdf_url` *(str)* — Ссылка на PDF-документ.

**Возвращает:**
- *(str)* — Сплошной текст из документа.

### `pdf.uni_cutter(pdf_url, uni_name)`
Возвращает обрезанный текст из PDF, оставляя только результаты обучения.

**Параметры:**
- `pdf_url` *(str)* — Ссылка на PDF-документ.
- `uni_name` *(str)* — Название университета (для корректного выделения блока).

**Возвращает:**
- *(str)* — Обработанный текст.

## Модуль: `rdt_database` as `db`

### `db.dataToDict(db_url)`
Возвращает словарь вида `{uni_name: uni_url}`, выгруженный из базы данных по ссылке.

**Параметры:**
- `db_url` *(str)* — Ссылка на базу данных PostgreSQL.

**Возвращает:**
- *(dict)* — Университеты и соответствующие им ссылки.

### `db.view(db_url)`
Открывает окно визуализации базы данных по ссылке, с возможностью выбирать таблицы.

**Параметры:**
- `db_url` *(str)* — Ссылка на базу данных PostgreSQL.

### Дополнительный модуль: `rdt_db_viewer`
Содержит описание класса `QTapplication`, используемого для визуализации содержимого базы данных.

---

## Модуль: `rdt_hhru` as `hh`

### `hh.job_req_search(search_term, exp_level='noExperience')`
Возвращает требования к вакансиям по заданной специальности (практически поискововой запрос), используя API hh.ru.

**Параметры:**
- `search_term` *(str)* — Название профессии или области поиска.
- `exp_level` *(str, optional)* — Уровень опыта (по умолчанию `'noExperience'`).

---

## Модуль: `rdt_ml` as `ml`

### `ml.Analyzer(skill_mapping={})`
Класс для анализа и сравнения требований к вакансиям и образовательных программ.

**Параметры:**
- `skill_mapping` *(dict)* — Словарь синонимов и схожих терминов, например: `{ "Python": ["python", "питон"] }`. Необходимо исключительно для анализа IT-специальностей.

### `analyzer.main_analyze(edu_texts, job_texts, metric='optimal')`
Выполняет сравнение программ обучения и требований из вакансий.

**Параметры:**
- `edu_texts` *(str)* — Текст образовательной программы.
- `job_texts` *(str)* — Текст вакансий или требований.
- `metric` *(str)* — Выбор метрики сравнения: `'optimal'`, `'cluster'`, `'semantic'`.
